# Тема 1.11 — Списки (list)

## Создание

В JS есть только один тип массива — `Array`. В Python `list` — это динамический массив объектов, ближайший аналог. Но с несколькими важными отличиями.

```python
# Создание
empty = []
numbers = [1, 2, 3]
mixed = [1, "two", 3.0, None, True]  # технически можно, но не стоит

# Конструктор
from_range = list(range(5))      # [0, 1, 2, 3, 4]
from_string = list("hello")      # ['h', 'e', 'l', 'l', 'o']
from_tuple = list((1, 2, 3))     # [1, 2, 3]

# Повторение — нет аналога в JS
zeros = [0] * 5    # [0, 0, 0, 0, 0]
```

**Ловушка:** `[0] * 5` копирует *ссылку*, а не объект — критично для вложенных списков:

```python
# НЕВЕРНО — все строки указывают на один и тот же список
matrix = [[0] * 3] * 3
matrix[0][0] = 99
print(matrix)  # [[99, 0, 0], [99, 0, 0], [99, 0, 0]] — сломано!

# ВЕРНО
matrix = [[0] * 3 for _ in range(3)]
matrix[0][0] = 99
print(matrix)  # [[99, 0, 0], [0, 0, 0], [0, 0, 0]]
```

## Индексация и срезы

Как в строках (тема 1.6) — всё то же самое, но элементы изменяемы.

```python
items = [10, 20, 30, 40, 50]

# Индексация
items[0]   # 10
items[-1]  # 50  ← отрицательные индексы — питоновская фишка

# Срезы [start:stop:step]
items[1:3]   # [20, 30]  — stop не включается
items[::-1]  # [50, 40, 30, 20, 10]  — разворот
items[::2]   # [10, 30, 50]  — каждый второй

# Срез возвращает новый список (не view)
copy = items[:]   # поверхностная копия
```

**Отличие от JS:** в JS нет встроенных срезов — только `.slice()`. В Python `[1:3]` работает везде: list, str, tuple, bytes.

## Изменение через срез

Это питоновская суперсила — нет аналога в JS:

```python
items = [1, 2, 3, 4, 5]

# Замена диапазона
items[1:3] = [20, 30]      # [1, 20, 30, 4, 5]

# Вставка без удаления
items[2:2] = [99, 100]     # [1, 20, 99, 100, 30, 4, 5]

# Удаление диапазона
items[1:3] = []            # удаляет элементы [1:3]

# Замена с другим количеством элементов
items = [1, 2, 3]
items[0:2] = [10, 20, 30]  # [10, 20, 30, 3] — заменяет 2 на 3 элемента
```

## Основные методы

| Метод | Что делает | JS-аналог |
|---|---|---|
| `append(x)` | добавить в конец | `push(x)` |
| `insert(i, x)` | вставить на позицию | `splice(i, 0, x)` |
| `extend(iterable)` | добавить все элементы | `push(...arr)` |
| `pop()` | удалить и вернуть последний | `pop()` |
| `pop(i)` | удалить и вернуть по индексу | `splice(i, 1)[0]` |
| `remove(x)` | удалить первое вхождение x | нет прямого аналога |
| `index(x)` | индекс первого вхождения | `indexOf(x)` |
| `count(x)` | количество вхождений | нет прямого аналога |
| `reverse()` | разворот на месте | `reverse()` |
| `sort()` | сортировка на месте | `sort()` |
| `copy()` | поверхностная копия | `[...arr]` |
| `clear()` | очистить список | `arr.length = 0` |

```python
items = [3, 1, 4, 1, 5, 9, 2, 6]

items.append(7)           # [3, 1, 4, 1, 5, 9, 2, 6, 7]
items.insert(0, 0)        # [0, 3, 1, 4, 1, 5, 9, 2, 6, 7]
items.extend([8, 9])      # добавляет 8 и 9 в конец

items.remove(1)           # удаляет ПЕРВУЮ единицу
items.index(9)            # → 5 (позиция в текущем списке)
items.count(1)            # → 1 (осталась одна единица после remove)

last = items.pop()        # удаляет и возвращает последний
third = items.pop(2)      # удаляет и возвращает items[2]
```

**Ловушка с `remove`:**

```python
items = [1, 2, 1, 3]
items.remove(1)   # удаляет первую 1, не все!
print(items)      # [2, 1, 3]

# Чтобы удалить все — используй comprehension:
items = [x for x in items if x != 1]
```

## sort() vs sorted()

Это важное Python-различие — нет прямого аналога в JS:

```python
nums = [3, 1, 4, 1, 5]

# sort() — изменяет список на месте, возвращает None
nums.sort()
print(nums)         # [1, 1, 3, 4, 5]

# sorted() — возвращает новый список, оригинал не трогает
original = [3, 1, 4]
new = sorted(original)
print(original)     # [3, 1, 4]  — не изменился
print(new)          # [1, 3, 4]

# Обратная сортировка
nums.sort(reverse=True)
sorted(nums, reverse=True)

# Сортировка по ключу
words = ["banana", "apple", "cherry"]
words.sort(key=len)         # по длине: ['apple', 'banana', 'cherry']
sorted(words, key=str.lower)  # регистронезависимо
```

**Подкапотно:** Python использует Timsort — гибрид merge sort и insertion sort, O(n log n). Стабильный — равные элементы сохраняют исходный порядок. В JS `Array.prototype.sort` тоже Timsort в V8 (но это деталь реализации).

**Ловушка:**

```python
result = nums.sort()   # классическая JS-привычка!
print(result)          # None — sort() ничего не возвращает
```

## Стек и очередь через list

```python
# Стек (LIFO) — list отлично подходит
stack = []
stack.append(1)   # push
stack.append(2)
stack.pop()       # pop → 2

# Очередь (FIFO) — list НЕ подходит для больших объёмов
queue = []
queue.append(1)
queue.pop(0)      # O(n) — сдвигает все элементы! Медленно.

# Правильно для очереди:
from collections import deque
queue = deque()
queue.append(1)
queue.popleft()   # O(1)
```

## Подкапотно

- `list` — динамический массив указателей на объекты (не сами объекты)
- При `append` Python выделяет память с запасом (~1.125x), чтобы избежать realloc на каждый вызов — амортизированное O(1)
- `insert(0, x)` — O(n), сдвигает все элементы вправо
- `pop()` — O(1), `pop(0)` — O(n)
- Каждый элемент списка — это указатель (8 байт на 64-bit), поэтому `mixed = [1, "two", None]` — нормально

```python
import sys
lst = []
for i in range(10):
    lst.append(i)
    print(f"len={len(lst)}, allocated_bytes={sys.getsizeof(lst)}")
# Увидишь скачки памяти — Python резервирует блоками
```

## Конспект: Списки (list)

### Суть
`list` — динамический массив указателей на объекты. Аналог JS `Array`, но с мощными срезами, сортировкой с ключом и чётким разделением in-place (`sort`, `reverse`) vs возвращающих новый объект (`sorted`, `reversed`, срезы). Главное отличие от JS: методы изменения не возвращают список — они возвращают `None`.

### JS → Python
| Python | Что делает | JavaScript |
|--------|-----------|------------|
| `lst.append(x)` | добавить в конец | `arr.push(x)` |
| `lst.extend(iter)` | добавить все элементы | `arr.push(...other)` |
| `lst.insert(i, x)` | вставить на позицию | `arr.splice(i, 0, x)` |
| `lst.pop()` | удалить и вернуть последний | `arr.pop()` |
| `lst.pop(i)` | удалить по индексу | `arr.splice(i, 1)[0]` |
| `lst.remove(x)` | удалить первое вхождение | нет аналога |
| `lst.index(x)` | индекс первого вхождения | `arr.indexOf(x)` |
| `x in lst` | проверка вхождения | `arr.includes(x)` |
| `lst.sort()` | сортировка на месте → `None` | `arr.sort()` → сам массив |
| `sorted(lst)` | новый отсортированный список | нет аналога |
| `lst[1:3]` | срез | `arr.slice(1, 3)` |
| `lst[:]` | поверхностная копия | `[...arr]` |
| `lst.copy()` | поверхностная копия | `[...arr]` |

### Синтаксис / API
```python
lst = [1, 2, 3]          # литерал
lst = list(range(5))     # из итерируемого
lst = [0] * 3            # [0, 0, 0] — только для примитивов!

lst[0]                   # первый элемент
lst[-1]                  # последний
lst[1:3]                 # срез [1, 2] (stop не включается)
lst[::-1]                # разворот — новый список
lst[::2]                 # каждый второй

lst.append(x)            # O(1) амортизированно
lst.insert(0, x)         # O(n) — медленно для начала списка
lst.extend([4, 5])       # O(k), k — длина добавляемого
lst.pop()                # O(1)
lst.pop(i)               # O(n)
lst.remove(x)            # удаляет первое x, ValueError если нет
lst.index(x)             # ValueError если x не найден
lst.count(x)             # количество вхождений x
lst.sort(key=fn, reverse=True)  # на месте, возвращает None
sorted(lst, key=fn)      # новый список
lst.reverse()            # на месте
lst.copy()               # поверхностная копия = lst[:]
lst.clear()              # очищает список
```

### Подкапотно
- `list` — массив указателей (8 байт каждый), не сами объекты
- При `append` CPython выделяет память с запасом (~1.125x от текущего размера) — amortized O(1)
- `insert(0, x)` и `pop(0)` — O(n): сдвигают все элементы
- Для очереди (FIFO) использовать `collections.deque` — `popleft()` O(1) вместо O(n)
- Timsort: стабильный, O(n log n) в худшем, O(n) для уже отсортированных данных

### Ловушки
```python
# 1. sort() возвращает None
result = [3,1,2].sort()   # result == None!
result = sorted([3,1,2])  # правильно

# 2. Вложенный список через умножение
bad = [[0]*3] * 3         # все строки — один объект!
bad[0][0] = 1             # меняет все строки
good = [[0]*3 for _ in range(3)]  # правильно

# 3. remove() удаляет только первое вхождение
[1, 2, 1].remove(1)       # → [2, 1], не [2]

# 4. pop(0) на больших списках — O(n)
from collections import deque
q = deque([1,2,3])
q.popleft()               # O(1)

# 5. Срез создаёт копию — изменение среза не меняет оригинал
lst = [1, 2, 3]
copy = lst[1:]
copy[0] = 99              # lst не изменился
```

### Когда использовать
- **list** — основная структура для упорядоченных изменяемых коллекций
- **deque** — очередь, двусторонний append/pop
- `lst[:]` или `lst.copy()` — поверхностная копия; `copy.deepcopy()` — глубокая
- `sorted()` вместо `.sort()` — когда нужен оригинал нетронутым (в функциях, пайплайнах)
- Не хранить разнородные данные в `list` по смыслу — для этого есть `tuple`, `dataclass`, `dict`

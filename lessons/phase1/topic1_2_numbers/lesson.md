## Целочисленная арифметика без переполнения

В JS `Number` — это `float64`, и при больших числах начинаются сюрпризы:

```js
// JS
Number.MAX_SAFE_INTEGER // 9007199254740991
9007199254740991 + 2   // 9007199254740992 — неправильно!
```

В Python `int` — **произвольная точность**, никакого переполнения:

```python
x = 9007199254740991 + 2
print(x)   # 9007199254740993 — точно

import math
print(math.factorial(100))
# 93326215443944152681699238856266700490715968264381621468592963895217599993229915608941463976156518286253697920827223758251185210916864000000000000000000000000
```

**Подкапотно:** Python хранит `int` как массив 30-битных "цифр" — число растёт динамически.
Числа от `-5` до `256` кэшируются как синглтоны (важно для `is`).

**Цена:** большие `int` медленнее C `long`. В числодробилках используют `numpy.int64`.

## Операторы деления: `/`, `//`, `%`

```python
# / — всегда возвращает float (даже если делится нацело)
10 / 2    # 5.0  ← не 5, а 5.0!
7 / 2     # 3.5

# // — целочисленное деление (floor division, округление ВНИЗ)
7 // 2    # 3
-7 // 2   # -4  ← ВАЖНО! не -3 как в JS

# % — остаток от деления (знак всегда как у делителя)
7 % 2     # 1
-7 % 2    # 1   ← в Python знак делителя
-7 % -2   # -1

# В JS: Math.trunc(-7 / 2) = -3, -7 % 2 = -1  ← знак делимого
```

**Правило:** `a == (a // b) * b + (a % b)` — всегда выполняется в Python.

**Пример использования** — конвертация секунд:

```python
seconds = 3725
hours   = seconds // 3600           # 1
minutes = (seconds % 3600) // 60    # 2
secs    = seconds % 60              # 5
print(f"{hours}h {minutes}m {secs}s")  # 1h 2m 5s
```

## Возведение в степень `**`

```python
2 ** 10     # 1024
2 ** 0.5    # 1.4142135623730951  (квадратный корень)
2 ** -1     # 0.5
(-2) ** 3   # -8
```

**Ловушка — порядок операций:**

```python
-2 ** 2    # -4  ← Python читает как -(2**2), не (-2)**2 !
(-2) ** 2  # 4   ← правильно, если нужен квадрат отрицательного
```

## float и его проблемы

Та же история, что и в JS — IEEE 754:

```python
0.1 + 0.2          # 0.30000000000000004
0.1 + 0.2 == 0.3   # False

# Правильное сравнение float:
import math
math.isclose(0.1 + 0.2, 0.3)  # True
```

## `Decimal` — точная арифметика

Для финансов и точных расчётов (аналог `decimal.js` в JS, но встроен в stdlib):

```python
from decimal import Decimal, ROUND_HALF_UP

# Строка, не float! float внесёт ошибку сразу при создании
price = Decimal("0.10")
tax   = Decimal("0.20")
total = price + tax
print(total)          # 0.30  ← точно!

total == Decimal("0.30")  # True

# Округление с нужным поведением
amount = Decimal("2.675")
rounded = amount.quantize(Decimal("0.01"), rounding=ROUND_HALF_UP)
print(rounded)  # 2.68
```

**Ловушка — не создавать Decimal из float:**

```python
Decimal(0.1)    # Decimal('0.1000000000000000055511151231257827021181583404541015625')
Decimal("0.1")  # Decimal('0.1')  ← правильно, только из строки
```

## `round()` — banker's rounding

```python
round(2.5)   # 2  ← не 3! (JS: Math.round(2.5) = 3)
round(3.5)   # 4
round(4.5)   # 4  ← не 5!
round(1.5)   # 2

# Python использует banker's rounding — к ближайшему чётному
# Уменьшает накопленную ошибку при массовых вычислениях
```

**Другие варианты округления:**

```python
import math

round(3.14159, 2)   # 3.14  — до N знаков
round(1234, -2)     # 1200  — отрицательная точность

math.floor(2.7)     # 2  — вниз (к -∞)
math.ceil(2.3)      # 3  — вверх (к +∞)
math.trunc(-2.7)    # -2 — к нулю (как JS parseInt)
```

## Конспект

### Суть
Python разделяет целые (`int`) и дробные (`float`) числа. `int` имеет произвольную точность — не переполняется никогда. `/` всегда возвращает `float`, для целочисленного деления — отдельный оператор `//`. `float` — IEEE 754, те же проблемы точности что в JS. Для финансов — только `Decimal` из строки.

### Python → JS

| Python | Что делает | JavaScript |
|--------|-----------|-----------|
| `int` без ограничений | целые числа произвольной точности | `Number` ограничен MAX_SAFE_INTEGER |
| `10 / 2 == 5.0` | деление всегда возвращает float | `10 / 2 === 5` |
| `10 // 3 == 3` | целочисленное деление (floor) | нет прямого аналога |
| `-7 // 2 == -4` | floor division — округление к −∞ | `Math.trunc(-7/2) === -3` (к нулю) |
| `-7 % 2 == 1` | остаток — знак совпадает с делителем | `-7 % 2 === -1` (знак делимого) |
| `2 ** 10` | возведение в степень | `Math.pow(2, 10)` / `2**10` (ES2016) |
| `round(2.5) == 2` | banker's rounding — к ближайшему чётному | `Math.round(2.5) === 3` |
| `from decimal import Decimal` | точная десятичная арифметика (stdlib) | `decimal.js` (npm) |

### Синтаксис / API

```python
# --- int ---
10 / 2      # 5.0  — ВСЕГДА float
10 // 3     # 3    — целочисленное деление (int)
10 % 3      # 1    — остаток
2 ** 10     # 1024 — степень
-7 // 2     # -4   — floor к -∞ (не -3!)
-7 % 2      # 1    — знак делителя

# --- float ---
import math
0.1 + 0.2               # 0.30000000000000004 — IEEE 754
0.1 + 0.2 == 0.3        # False!
math.isclose(0.1 + 0.2, 0.3)           # True — правильное сравнение
math.isclose(a, b, rel_tol=1e-9)       # с точностью

float('inf')            # бесконечность
float('-inf')           # минус бесконечность
float('nan')            # NaN
math.isnan(float('nan'))  # True — правильная проверка NaN

# --- round ---
round(2.5)              # 2  — banker's rounding!
round(3.5)              # 4
round(3.14159, 2)       # 3.14 — до N знаков после запятой
round(1234, -2)         # 1200 — до сотен

math.floor(2.7)         # 2  — вниз (к -∞)
math.ceil(2.3)          # 3  — вверх (к +∞)
math.trunc(-2.7)        # -2 — к нулю (как JS parseInt)

# --- Decimal ---
from decimal import Decimal, ROUND_HALF_UP

price = Decimal("19.99")        # ТОЛЬКО из строки!
tax   = Decimal("0.20")
total = price * (1 + tax)       # Decimal * (1 + Decimal)

# Decimal дружит с int, но не с float
price * 2                       # OK — Decimal
price * 2.0                     # TypeError!

# Округление с контролем направления
amount = Decimal("2.675")
amount.quantize(Decimal("0.01"), rounding=ROUND_HALF_UP)  # 2.68

# Обратно в float
float(price)                    # 19.99
```

### Подкапотно
- `int` хранится как массив 30-битных "цифр" — растёт в памяти динамически. Числа от -5 до 256 — синглтоны (кэшируются).
- `float` — IEEE 754 double precision (64 бит), как JS `Number`. `0.1` в двоичной системе не представимо точно — отсюда погрешности.
- `//` на отрицательных: `floor(-3.5) = -4` — Python всегда округляет к минус бесконечности. JS `Math.trunc` — к нулю. Разное поведение!
- `Decimal` хранит число как `(знак, цифры, экспонент)` в десятичной системе. `0.1` точен. Стоит ~50x медленнее float.

### Ловушки

```python
# 1. Деление всегда float — даже если делится нацело
type(10 / 2)   # <class 'float'>, не int!
10 / 2         # 5.0, не 5

# 2. floor division на отрицательных
-7 // 2    # -4 — не -3! (Python: к -∞, JS: к нулю)
7 // -2    # -4 — тоже floor

# 3. round — banker's rounding
round(2.5)  # 2 (не 3!)
round(4.5)  # 4 (не 5!)
round(3.5)  # 4 — к чётному

# 4. Decimal только из строки
from decimal import Decimal
Decimal(0.1)    # Decimal('0.1000000000000000055511...')  — уже с погрешностью!
Decimal("0.1")  # Decimal('0.1')  — точно

# 5. float нельзя сравнивать через ==
0.1 + 0.2 == 0.3   # False — ВСЕГДА используй math.isclose()

# 6. Степень отрицательного числа
-2 ** 2    # -4  (Python читает как -(2**2))
(-2) ** 2  # 4   — нужны скобки
```

### Когда использовать

- `int` — счётчики, индексы, ID, пагинация, любая целочисленная арифметика
- `float` — координаты, проценты, физические величины (где небольшая погрешность ок)
- `Decimal` — деньги, налоги, финансы — любой расчёт где погрешность недопустима
- `math.isclose()` — сравнение `float` вместо `==` (в тестах, бизнес-логике)
- `//` и `%` — пагинация (`page = offset // limit`), конвертация времени, проверка делимости (`n % 2 == 0`)
- `math.floor/ceil/trunc` — когда нужен явный контроль направления округления

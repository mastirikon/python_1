# 1.14 — Множества (`set`, `frozenset`)

## Создание

```python
s = {1, 2, 3}
s = set([1, 2, 2, 3])  # {1, 2, 3} — дубли удаляются
s = set()              # пустое множество — НЕ {}, это dict!

# Элементы должны быть hashable — как ключи dict
{1, "hello", (1, 2)}   # OK
{[1, 2]}               # TypeError: unhashable type: 'list'
```

**JS-аналогия:** `new Set([1, 2, 3])`. Но в Python `set` поддерживает математические операции через операторы (`|`, `&`, `-`, `^`).

**Подкапотно:** `set` — хэш-таблица без значений (только ключи). Поэтому `in` — O(1), как в `dict`. Порядок элементов не гарантирован.

---

## Основные операции

```python
s = {1, 2, 3}

s.add(4)        # добавить элемент
s.remove(2)     # удалить — KeyError если нет
s.discard(99)   # удалить — безопасно, нет KeyError
s.pop()         # удалить и вернуть случайный элемент

2 in s          # True — O(1)
len(s)          # количество элементов
```

---

## Математические операции

```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b    # объединение        → {1, 2, 3, 4, 5, 6}
a & b    # пересечение        → {3, 4}
a - b    # разность           → {1, 2}  (есть в a, нет в b)
a ^ b    # симметр. разность  → {1, 2, 5, 6}

# Операторы присваивания
a |= b   # a.update(b)
a &= b   # a.intersection_update(b)
a -= b   # a.difference_update(b)
```

## Отношения между множествами

```python
{1, 2} <= {1, 2, 3}           # True — подмножество
{1, 2} <  {1, 2, 3}           # True — строгое подмножество
{1, 2, 3} >= {1, 2}           # True — надмножество

{1, 2}.issubset({1, 2, 3})    # True
{1, 2, 3}.issuperset({1, 2})  # True
{1, 2}.isdisjoint({3, 4})     # True — нет общих элементов
```

## frozenset

Иммутабельная версия `set` — можно использовать как ключ `dict` или положить в другой `set`.

```python
fs = frozenset([1, 2, 3])
fs.add(4)                    # AttributeError — иммутабельный

d = {frozenset([1, 2]): "pair"}  # OK как ключ dict
```

## Типичные паттерны

```python
# Дедупликация
unique = list(set([1, 2, 2, 3, 3, 3]))  # [1, 2, 3]

# Быстрая проверка принадлежности — O(1) vs O(n) у списка
allowed = {"admin", "moderator", "editor"}
if user.role in allowed:
    ...

# Общие элементы
tags_a = {"python", "django", "api"}
tags_b = {"python", "fastapi", "api"}
common = tags_a & tags_b   # {"python", "api"}

# Только в одном из двух
only_a = tags_a - tags_b   # {"django"}
```

## Конспект

### Суть
`set` — неупорядоченная коллекция уникальных hashable элементов. Основан на хэш-таблице — проверка `in` работает за O(1). Поддерживает математические операции с множествами через операторы. `frozenset` — иммутабельная версия, может быть ключом словаря.

### Python | Что делает | JavaScript
| Python | Что делает | JavaScript |
|--------|-----------|-----------|
| `{1, 2, 3}` | создать множество | `new Set([1, 2, 3])` |
| `set()` | пустое множество | `new Set()` |
| `s.add(x)` | добавить элемент | `s.add(x)` |
| `s.remove(x)` | удалить, KeyError если нет | `s.delete(x)` (не бросает) |
| `s.discard(x)` | удалить безопасно | `s.delete(x)` |
| `x in s` | проверка O(1) | `s.has(x)` |
| `a \| b` | объединение | нет оператора, `new Set([...a, ...b])` |
| `a & b` | пересечение | нет оператора |
| `a - b` | разность | нет оператора |
| `a ^ b` | симметричная разность | нет оператора |
| `frozenset` | иммутабельное множество | нет аналога |

### Ловушки

```python
# {} — это dict, не set!
s = {}        # dict
s = set()     # set

# Элементы должны быть hashable
{[1, 2]}      # TypeError: unhashable type: 'list'
{(1, 2)}      # OK

# Порядок не гарантирован
s = {3, 1, 2}
list(s)       # может быть [1, 2, 3] или другой порядок

# remove vs discard
s = {1, 2}
s.remove(99)  # KeyError
s.discard(99) # OK
```

### Когда использовать
- **Дедупликация** — `list(set(items))`
- **Проверка принадлежности** — `x in allowed_set` вместо `x in list` (O(1) vs O(n))
- **Операции с коллекциями** — пересечение, разность, объединение тегов/прав/ID
- **frozenset** — как ключ словаря, например кэш по набору параметров

# Тема 1.12 — Кортежи (tuple)

## Что такое tuple

`tuple` — иммутабельный список. Как `list`, но изменить нельзя. В JS прямого аналога нет — ближе всего `Object.freeze([1, 2, 3])`, но это хак. В Python это первоклассный тип.

```python
point = (10, 20)
rgb = (255, 128, 0)
single = (42,)   # ВАЖНО: запятая обязательна для tuple из одного элемента
empty = ()

# Скобки часто необязательны — tuple создаёт запятая, не скобкиkv
point = 10, 20        # тоже tuple
a, b, c = 1, 2, 3    # распаковка — тоже tuple под капотом
```

**Ловушка — одноэлементный tuple:**
```python
x = (42)    # это int, не tuple! Скобки — просто группировка
x = (42,)   # вот это tuple
type((42))  # <class 'int'>
type((42,)) # <class 'tuple'>
```

---

## Иммутабельность

```python
point = (10, 20)
point[0] = 99   # TypeError: 'tuple' object does not support item assignment

# Но если внутри tuple лежит изменяемый объект — он изменится:
t = ([1, 2], [3, 4])
t[0].append(99)   # OK!
print(t)          # ([1, 2, 99], [3, 4])
# tuple не изменился (ссылки те же), но объект внутри — да
```

---

## Индексация и срезы

Всё то же, что у `list`:

```python
t = (10, 20, 30, 40, 50)
t[0]      # 10
t[-1]     # 50
t[1:3]    # (20, 30) — возвращает tuple
t[::-1]   # (50, 40, 30, 20, 10)
```

---

## Методы

У `tuple` только два метода (изменять нельзя, поэтому и методов мало):

```python
t = (1, 2, 3, 2, 1)
t.count(2)   # 2 — количество вхождений
t.index(3)   # 2 — индекс первого вхождения (ValueError если нет)
```

## Распаковка (destructuring)

Это как деструктуризация в JS, но мощнее:

```python
# Базовая распаковка
x, y = (10, 20)
r, g, b = (255, 128, 0)

# С остатком — аналог JS rest
first, *rest = (1, 2, 3, 4, 5)
# first = 1, rest = [2, 3, 4, 5]  ← rest всегда list!

*head, last = (1, 2, 3, 4, 5)
# head = [1, 2, 3, 4], last = 5

first, *middle, last = (1, 2, 3, 4, 5)
# first = 1, middle = [2, 3, 4], last = 5

# Вложенная распаковка
(a, b), c = (1, 2), 3
point = ((10, 20), (30, 40))
(x1, y1), (x2, y2) = point

# Игнорирование значений — конвенция _
_, important, _ = (1, 42, 3)
```

**Swap без temp — питоновская идиома:**
```python
a, b = 1, 2
a, b = b, a   # под капотом: создаётся tuple (b, a), потом распаковывается
```

## Когда tuple, а когда list

| Ситуация | Что использовать |
|---|---|
| Коллекция однородных данных (список юзеров) | `list` |
| Запись с фиксированной структурой (координата, RGB) | `tuple` |
| Ключ словаря | `tuple` (list нельзя — не hashable) |
| Возврат нескольких значений из функции | `tuple` |
| Данные не должны меняться | `tuple` |

```python
# tuple как ключ dict — list так не умеет
locations = {(55.75, 37.61): "Москва", (59.93, 30.31): "Питер"}

# Возврат нескольких значений
def min_max(items):
    return min(items), max(items)  # возвращает tuple

lo, hi = min_max([3, 1, 4, 1, 5])
```

## namedtuple

Обычный `tuple` не даёт именам — только индексы. `namedtuple` добавляет имена полей:

```python
from collections import namedtuple

Point = namedtuple("Point", ["x", "y"])
p = Point(10, 20)

p.x      # 10 — доступ по имени
p[0]     # 10 — доступ по индексу тоже работает
p.x, p.y = 99, 0  # TypeError — всё ещё иммутабельный

# Можно распаковывать
x, y = p

print(p)   # Point(x=10, y=20) — красивый repr
```

---

## typing.NamedTuple — современный способ

```python
from typing import NamedTuple

class Point(NamedTuple):
    x: float
    y: float
    label: str = ""   # default значение

p = Point(10.0, 20.0)
p = Point(x=1.0, y=2.0, label="origin")

# Все плюсы namedtuple + type hints + default values
```

**Отличие от `dataclass`:** `NamedTuple` иммутабельный и является tuple (можно распаковывать, использовать как ключ dict). `dataclass` — обычный изменяемый класс.

---

## Подкапотно

- `tuple` хранится как фиксированный массив указателей — без оверхеда на динамическое расширение
- Меньше памяти чем `list`:
```python
import sys
sys.getsizeof((1, 2, 3))   # 72 байта
sys.getsizeof([1, 2, 3])   # 88 байт
```
- `tuple` — hashable (если все элементы hashable), поэтому может быть ключом `dict` и элементом `set`
- CPython кэширует маленькие tuple (как маленькие int) — `(1, 2)` может быть одним объектом в памяти

## Конспект: Кортежи (tuple)

### Суть
`tuple` — иммутабельная последовательность. Не readonly-массив (как `Object.freeze`), а полноценный тип. Tuple создаёт **запятая**, не скобки. Используется для записей с фиксированной структурой, ключей словарей, возврата нескольких значений из функции.

### JS → Python
| Python | Что делает | JavaScript |
|--------|-----------|------------|
| `(1, 2, 3)` | кортеж | нет аналога (≈ `Object.freeze([1,2,3])`) |
| `x, y = t` | распаковка | `const [x, y] = arr` |
| `first, *rest = t` | с остатком | `const [first, ...rest] = arr` |
| `a, b = b, a` | swap | `[a, b] = [b, a]` |
| `t.count(x)` | количество вхождений | нет аналога |
| `t.index(x)` | индекс первого вхождения | `arr.indexOf(x)` |
| `namedtuple` / `NamedTuple` | tuple с именами полей | нет аналога |

### Синтаксис / API
```python
t = (1, 2, 3)         # литерал
t = 1, 2, 3           # скобки необязательны
t = (42,)             # одноэлементный — запятая обязательна!
t = ()                # пустой

t[0]                  # индексация
t[-1]                 # последний
t[1:3]                # срез — возвращает tuple

t.count(x)            # количество вхождений x
t.index(x)            # первый индекс x, ValueError если нет

x, y = t              # распаковка
first, *rest = t      # rest — всегда list
_, val, _ = t         # игнорирование через _

# namedtuple
from collections import namedtuple
Point = namedtuple("Point", ["x", "y"])
p = Point(10, 20); p.x; p.y

# NamedTuple (современный способ)
from typing import NamedTuple
class Point(NamedTuple):
    x: float
    y: float = 0.0
```

### Подкапотно
- Фиксированный массив указателей — нет оверхеда на динамическое расширение
- Меньше памяти чем `list` (72 vs 88 байт для 3 элементов)
- Hashable (если элементы hashable) → может быть ключом `dict` и элементом `set`
- CPython кэширует маленькие tuple аналогично малым int

### Ловушки
```python
# 1. Одноэлементный tuple
x = (42)    # int, не tuple!
x = (42,)   # tuple — нужна запятая

# 2. Иммутабельность не глубокая
t = ([1, 2], [3, 4])
t[0].append(99)   # OK — список внутри изменился
# tuple не изменился (ссылки те же)

# 3. *rest всегда list, не tuple
first, *rest = (1, 2, 3)
type(rest)   # <class 'list'>
```

### Когда использовать
- **tuple** — запись с фиксированной структурой: координата, RGB, диапазон, пара ключ-значение
- **tuple как ключ dict** — когда нужен составной ключ: `{(x, y): value}`
- **возврат нескольких значений** — `return min_val, max_val` (питоновская идиома)
- **NamedTuple** — когда нужны имена полей + иммутабельность + type hints; легче dataclass если не нужны методы
- **list** — однородные коллекции, которые будут изменяться

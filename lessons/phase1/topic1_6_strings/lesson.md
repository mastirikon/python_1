## 1. Строки в Python — базовая природа

### Иммутабельность

**JS-аналогия:** строки в JS тоже иммутабельны — изменить символ нельзя, только создать новую строку. Python — то же самое, но более явно.

```python
s = "hello"
s[0] = "H"  # TypeError: 'str' object does not support item assignment
```

В JS это молча ничего не делает. В Python — явная ошибка.

**Подкапотно:** строка в Python — объект типа `str`, хранится как массив Unicode-кодпоинтов (UTF-32 внутри для произвольного доступа по индексу за O(1)). Начиная с Python 3.3 — оптимизированное хранение: Latin-1 / UCS-2 / UCS-4 в зависимости от контента.

**Используется:** строки передаются в функции безопасно — никто их не изменит. Можно использовать как ключи словаря и элементы множеств (они хешируемы).

---

### Создание строк

```python
s1 = 'одинарные кавычки'
s2 = "двойные кавычки"
s3 = '''многострочная
строка'''
s4 = """тоже
многострочная"""

# Raw-строки — экранирование отключено (для regex, Windows-путей)
s5 = r"C:\Users\anton\file.txt"  # \U и \f не интерпретируются
s6 = r"\n — это не перенос строки"

# Bytes — не str, отдельный тип
b = b"hello"        # bytes, не str
type(b)             # <class 'bytes'>
```

**Ловушка из JS:** в JS нет разницы между `'` и `"`. В Python тоже нет — выбирай любой стиль и придерживайся его. Но `"""` — именно для многострочных строк, не для "важных" строк.

## 2. Индексация

### Прямая и обратная

```python
s = "python"
#    0 1 2 3 4 5   — прямые индексы
#   -6-5-4-3-2-1   — обратные индексы

s[0]   # 'p'
s[5]   # 'n'
s[-1]  # 'n'  ← последний символ
s[-2]  # 'o'
s[-6]  # 'p'
```

**JS-аналогия:** `s[-1]` в JS — `undefined` (или `s.at(-1)` в современном JS). В Python отрицательные индексы — встроены в язык и работают везде: строки, списки, кортежи.

**Подкапотно:** `s[-1]` — это синтаксический сахар для `s[len(s) - 1]`. Python сам вычитает из длины.

**Ловушка:** `s[len(s)]` — IndexError. В JS `s[s.length]` — просто `undefined`. Python строже.

```python
s = "python"
s[6]   # IndexError: string index out of range
s[-7]  # IndexError: string index out of range
```

## 3. Срезы (slices)

### Базовый синтаксис

```python
s = "python"

s[1:4]    # 'yth'  — от индекса 1 до 4 (не включая 4)
s[2:]     # 'thon' — от 2 до конца
s[:3]     # 'pyt'  — от начала до 3 (не включая 3)
s[:]      # 'python' — копия всей строки
```

**JS-аналогия:** `s.slice(1, 4)` — то же самое. Отличие: в Python квадратные скобки, синтаксис встроен в язык, а не метод.

**Правило:** `s[start:stop]` — символы с индекса `start` до `stop - 1` включительно. То есть длина среза = `stop - start`.

---

### Шаг (step)

```python
s = "python"

s[::2]    # 'pto'  — каждый второй символ
s[1::2]   # 'yhn'  — каждый второй, начиная с 1
s[::-1]   # 'nohtyp' — строка наоборот!
s[5:1:-1] # 'nohт' — с 5 до 2 (не включая 1), шаг -1
```

**`s[::-1]` — питоновский идиом для разворота строки.** В JS — `s.split('').reverse().join('')`.

**Подкапотно:** срез создаёт новый объект `str`. Это O(k), где k — длина среза, не O(1). Для строк нет "view" как в numpy.

---

### Срезы с отрицательными индексами

```python
s = "python"

s[-3:]    # 'hon' — последние 3 символа
s[:-2]    # 'pyth' — всё кроме последних 2
s[-4:-1]  # 'tho' — с -4 до -2 включительно
```

**Ловушка:** срезы никогда не кидают IndexError — даже если индексы выходят за границы:

```python
s = "python"
s[100:]   # '' — пустая строка, не ошибка
s[:1000]  # 'python' — вся строка
```

Это отличается от обычной индексации! Срезы "мягкие", индексы "строгие".

## 4. Встроенные функции для строк

```python
s = "python"

len(s)       # 6 — количество символов
min(s)       # 'h' — минимальный символ по Unicode
max(s)       # 'y' — максимальный символ по Unicode

# Перебор строки как итерируемого
for ch in s:
    print(ch)   # p, y, t, h, o, n

# Проверка вхождения (O(n))
"yt" in s    # True
"yz" in s    # False
```

**JS-аналогия:** `s.includes("yt")` — то же самое. В Python — оператор `in`, работает для любых итерируемых.

## 5. str vs bytes

```python
# str — Unicode текст (Python 3)
s = "привет"
type(s)   # <class 'str'>
len(s)    # 6 — символов, не байт

# bytes — сырые байты
b = "привет".encode("utf-8")
type(b)   # <class 'bytes'>
len(b)    # 12 — байт (каждый кириллический символ = 2 байта в UTF-8)

# Обратно
b.decode("utf-8")  # 'привет'
```

**JS-аналогия:** в Node.js `Buffer` vs `string`. В Python — `bytes` vs `str`. В Python 3 они строго разделены — нельзя смешивать:

```python
"hello" + b"world"  # TypeError: can only concatenate str (not "bytes") to str
```

**Ловушка из Python 2:** в Python 2 `str` — это байты, `unicode` — текст. В Python 3 всё иначе: `str` — всегда Unicode. Если видишь старый код — учитывай это.

**Используется:** bytes — при работе с сетью, файлами в бинарном режиме, шифровании. Str — для всего текстового.

## 6. Конкатенация и повторение

```python
# Конкатенация
"hello" + " " + "world"   # 'hello world'

# Повторение
"ha" * 3    # 'hahaha'
"-" * 20    # '--------------------' (разделитель)

# Ловушка: конкатенация в цикле — O(n²)
result = ""
for i in range(1000):
    result += str(i)    # Каждый раз создаёт новую строку!

# Правильно: join
result = "".join(str(i) for i in range(1000))  # O(n)
```

**JS-аналогия:** в JS движки оптимизируют `+=` для строк. В Python — нет гарантий. `"".join(...)` — питоновский идиом для сборки строк из частей.

**Подкапотно:** CPython иногда оптимизирует `+=` когда строка имеет один владелец (`refcount == 1`), но на это нельзя полагаться. `join` — всегда O(n).

## Конспект: Строки в Python

### Суть
Строки в Python — иммутабельные последовательности Unicode-символов. В отличие от JS, изменение по индексу кидает `TypeError`, а не молчит. Срезы `s[start:stop:step]` встроены в синтаксис языка и работают мягко (без IndexError). `str` и `bytes` — строго разные типы, смешивать нельзя.

### Python → JS

| Python | Что делает | JavaScript |
|--------|-----------|-----------|
| `s[-1]` | последний символ | `s[s.length - 1]` или `s.at(-1)` |
| `s[1:4]` | срез — синтаксис языка | `s.slice(1, 4)` |
| `s[::-1]` | реверс строки | `s.split('').reverse().join('')` |
| `s[len(s)]` → `IndexError` | индекс строже — кидает ошибку | `s[s.length]` → `undefined` |
| `s[100:]` → `''` | срез мягче — никогда не кидает ошибку | нет аналога |
| `"x" in s` | проверка вхождения подстроки | `s.includes("x")` |
| `bytes` | сырые байты | `Buffer` |
| `"".join(list)` | сборка строки — всегда O(n) | `arr.join("")` |

### Синтаксис / API
```python
s = "python"

# Индексация
s[0]       # 'p' — первый
s[-1]      # 'n' — последний

# Срезы
s[1:4]     # 'yth' — с 1 до 3 включительно
s[:3]      # 'pyt' — от начала
s[2:]      # 'thon' — до конца
s[::-1]    # 'nohtyp' — реверс
s[::2]     # 'pto' — каждый второй
s[-3:]     # 'hon' — последние 3

# Длина и итерация
len(s)         # 6
for ch in s:   # итерация по символам
    ...
"yt" in s      # True — проверка вхождения

# str <-> bytes
s.encode("utf-8")         # str → bytes
b.decode("utf-8")         # bytes → str

# Сборка строк
"".join(["a", "b", "c"])  # 'abc' — правильный способ
```

### Подкапотно
- `str` хранится как массив Unicode-кодпоинтов. Python 3.3+ использует compact representation: Latin-1 (1 байт/символ), UCS-2 (2 байт), UCS-4 (4 байт) — в зависимости от максимального символа в строке.
- Отрицательный индекс `s[-k]` → `s[len(s) - k]` — вычисляется интерпретатором.
- Срез создаёт новый объект `str`, O(k) по длине среза.
- `+=` в цикле: каждая операция может создавать новую строку — O(n²) суммарно. `join` — O(n).
- Строки хешируемы → можно использовать как ключи `dict` и элементы `set`.

### Ловушки
```python
# 1. Индекс за границей — ошибка, срез — нет
s = "hi"
s[5]    # IndexError
s[5:]   # '' — пустая строка, всё ок

# 2. Изменение по индексу
s[0] = "H"  # TypeError — нельзя!

# 3. Конкатенация в цикле
result = ""
for word in words:
    result += word   # O(n²) — плохо
result = "".join(words)  # O(n) — хорошо

# 4. str и bytes нельзя смешивать
"text" + b"bytes"   # TypeError

# 5. len() считает символы, не байты
len("привет")              # 6 (символов)
len("привет".encode())     # 12 (байт в UTF-8)
```

### Когда использовать
- `s[::-1]` — реверс строки (интервью, парсинг)
- `s[-n:]` — последние n символов (хвосты логов, расширения файлов)
- `s[1:-1]` — убрать первый и последний символ (убрать кавычки/скобки)
- `"sep".join(list)` — сборка строки из списка (всегда вместо `+=` в цикле)
- `bytes` / `encode` / `decode` — сеть, файлы, шифрование
- `r"..."` — regex паттерны, Windows-пути
